# SpamFilter Pro

**GitHub-Styled ML Classifier with Streamlit UI**

SpamFilter Pro is a single-page academic machine learning app for SMS spam detection. It uses the UCI SMS Spam Collection dataset, a TF-IDF vectorizer, and Multinomial Naive Bayes to classify pasted messages as `[SPAM]` or `[HAM]` with a confidence score and one-click clipboard copy.

The product is intentionally simple: no login, no database, no charts, no gradients, no deep learning stack. Just a fast classical ML classifier wrapped in a polished GitHub-dark Streamlit interface.

## Features

- GitHub dark-mode visual system using Primer-inspired colors.
- Monospace code-block layout with a flat, lightweight UI.
- SMS spam classification using `TfidfVectorizer` and `MultinomialNB`.
- Automatic artifact loading, with local training fallback when model files are missing.
- Confidence score for every prediction.
- One-click result copying through inline vanilla JavaScript.
- Character counter, empty-input guard, and 10,000-character input cap.
- Mobile-responsive Streamlit layout.

## Dataset

Dataset: **SMS Spam Collection Dataset**

Source: UCI Machine Learning Repository / Kaggle mirrors

- Rows: 5,574 SMS messages
- Labels: `ham`, `spam`
- Approximate balance: 87% ham / 13% spam
- License: Creative Commons Attribution 4.0 International via UCI
- Citation: Almeida, T. & Hidalgo, J. (2011). SMS Spam Collection [Dataset]. UCI Machine Learning Repository. https://doi.org/10.24432/C5CC84

This repo includes the converted UTF-8 CSV file:

```text
spam.csv
```

Expected columns:

```csv
label,message
ham,Go until jurong point...
spam,Free entry in 2 a wkly comp...
```

The original downloaded archive and source text files live in `data/` for traceability.

## Quick Start

Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Train the model:

```bash
python3 train_model.py
```

Run the app:

```bash
streamlit run app.py
```

Open the local Streamlit URL shown in the terminal, usually:

```text
http://localhost:8501
```

## Model Training

The training script:

1. Loads `spam.csv`.
2. Validates the `label` and `message` columns.
3. Builds TF-IDF features with unigrams and bigrams.
4. Trains `MultinomialNB`.
5. Evaluates with a stratified 80/20 test split.
6. Runs 5-fold cross-validation.
7. Saves:
   - `models/spam_model.pkl`
   - `models/vectorizer.pkl`
   - `models/metrics.pkl`

Run:

```bash
python3 train_model.py
```

Example output:

```text
SpamFilter Pro training complete
Rows: 5,574
Ham / Spam: 4,827 / 747
Test accuracy: 98%+
5-fold CV accuracy: 98%+
```

Exact scores may vary slightly by dependency version, but the target is at least 97% accuracy.

## Clipboard Implementation

The copy feature uses:

```python
streamlit.components.v1.html(...)
```

The embedded component calls:

```javascript
navigator.clipboard.writeText(resultText)
```

It also includes a textarea fallback for browsers that restrict the Clipboard API. The green `✓ Copied!` feedback fades after 1.5 seconds using pure CSS keyframes.

## Limitations

- English-only v1.
- SMS-length messages are the intended domain.
- No history logging or user accounts.
- No online model updates.
- Confidence is the model probability, not a formal guarantee of correctness.
